二分搜索算法 Binary Search

下载 binarySearch.py

二分搜索是一种在有序排列的列表中搜索目标的算法，通过重复判断目标在哪一半序列中来实现。图书馆的书通常都有编号并且按编号摆放，知道编号以后找书“最”均等的做法就是找到书所在区域的中间那个书架的中间一层的中间那本书，然后判断你要找的书在这之前还是在这之后。

然后重复类似的过程：从你选中的那一半的大概中间位置开始，判断你要找的书是在前一半还是后一半。重复这个过程，直到找到这本书，或者找到了位置但是书已经被借走了。

（译注：对于不了解数据结构的读者来说，可能需要提醒一下，二分搜索仅适用于“有序排列”的列表，简化来说就是符合某种从小到大或者从大到小的顺序，因此在列表中存入新的内容时，需要花费额外的操作找到保存位置。 所以虽然看起来二分查找比一些其他搜索算法效率高出许多，但也需要考虑到算法把一部分计算负担转移到了插入过程，也需要考虑应用场景和需求的不同。 对于实现过程中如何找到序列的中间位置，也有很多方法，比如使用二叉树来保存序列。 算法有效率高的，有效率低的，但很少有能像魔法一样简化问题的。）

组合 Combinations

（译注：指排列组合当中的组合）

下载 combinations.py

给定一系列元素，我们可以得到这些元素的不同组合方式。例如，指定4个字母，ABCD，不同元素数量的组合（k-combinations）为：

  • 0个元素的组合（0-combinations）和4个元素的组合（4-combinations）都只有1种：分别是空字符串和ABCD。
  • 1个元素的组合（1-combinations）和3个元素的组合（3-combinations）都有4种：分别是 A, B, C, D 和 ABC, ABD, ACD, BCD。
  • 2个元素的组合（2-combinations）最多，有6种：AB, AC, AD, BC, BD 和 CD。

译注：下面代码的思路是，当我们需要在集合chars中找k个元素的组合时，可以将问题分为两种情况。 第一个元素包含在组合中，和第一个元素不包含在组合中。 组合包含第一个元素，则需要在剩下的元素中找出所有k-1个元素的组合，再和第一个元素结合在一起，组成k个元素的组合。 组合不包含第一个元素，则在去掉第一个元素的集合中找k个元素的组合。 将两种情况相加，就是全部的k个元素的组合。
在第一种情况中，问题从找k个元素的组合，变成了找k-1个元素的组合。 在第二种情况中，问题从在整个集合中找组合，变成了在去掉第一个元素的集合中找组合。 两种情况下问题的规模都缩小了，而想要确保递归有解，关键就在于保证递归有“出口”，并且每一步问题的规模都在缩小。
还有一种思路，可能能让我们更方便的理解为什么组合问题，是可以用递归来解决的。让我们从寻找1个元素的组合逐渐向上递推。
1个元素的组合，就是简单的取出集合中的每一个元素。
2个元素的组合，可以理解为先确定1个元素，再从剩余元素中取1个元素，形成组合。（为避免重复，可以将集合替换成序列，后选取的元素只能在先选取的元素的位置的后面，呃······dddd，摆烂）
3个元素的组合，相当于先确定的2个元素组合，再从剩余元素中取1个。
递归已经很明显了，想确定3个元素的组合，需要先确定2个元素的组合，而确定2个元素的组合，需要先确定1个元素的组合。以此类推，想确定k个元素的组合，需要先找到k-1个元素的组合。

计算斐波那契数列的数字，是带有递归属性的。例如，如果我们需要计算第10个斐波那契数字，需要把第9个和第8个斐波那契数字相加。要计算第9个和第8个，又需要分别计算第8个、第7个相加和第7个、第6个相加。这将带来很多重复的计算：注意计算第10个和第9个斐波那契数字时都需要用到第8个。这种递归会一直持续到递归出口，即第1个和第2个斐波那契数字，而这两个数字被指定为都是1。

归并排序 Merge Sort

下载 mergeSort.py

计算机科学家冯·诺伊曼（John von Neumann）在 1945 年发明了归并排序算法，它是一种高效的、通用的排序算法，并且特别适用于链表的。
这种算法使用了分治法策略：每次对归并排序函数 mergeSort() 的调用都将未排序的列表一分为二，直到它们被分割为长度仅为 0 或 1 的小段。 然后，随着递归调用的返回，这些小段会被逐渐合并成有序的整体（归并操作）。当最后一层的递归调用返回的时候，整个列表完成了排序。

（译注：归并操作，是归并排序的一个步骤，它的功能是用两个标记（指针）来把两个有序的列表合并为一个更长的有序列表。
方法是
1. 先把两个标记分别放在两个小的有序列表的开始位置
2. 比较两个标记所指向的元素的大小，选出大的，放在合并的大列表里
3. 把刚刚取出元素的小列表的标记向后移。
4. 重复步骤2、3直到所有元素都进入大列表）

举例来说，按步骤将列表 [0, 7, 6, 3, 1, 2, 5, 4]，分为两段 [0, 7, 6, 3] 和 [1, 2, 5, 4]，并且分别对这两段进行递归调用。 不断分割后，原本有 8 个元素的列表，最终会变成 8 个分别有 1 个元素的小列表，只有单一元素的列表或者没有元素的列表是不需要排序的，它们是天然有序的，这样我们就得到了 8 个小的有序列表。
然后随着递归调用的“返回”，进行归并操作，两个有 1 个元素的小列表变成了一个有 2 个元素的“有序”列表。 两个有 2 个元素的有序列表被合并成一个有 4 个元素的有序列表。以此类推，直到合并为 8 个元素的有序列表。 我们就完成了对原本列表的排序。

快排序 Quicksort

下载 quicksort.py

快排序使用了分治策略当中的分区、分块。 可以这样来理解分块： 想象你有一书架的书，并且没有有效的索引，所有的书都散乱的摆放着，而现在你想按书名的首字母为书架上的书排序。 这意味着你需要重复，取其中的一本放在书架上正确的位置，经过很长的一段时间来重新安排位置，直到每一本都排好。 而如果先是把所有书都取出来，按从 A 到 M、从 N 到 Z 分成两堆，也许会降低思考难度，节约时间（在这个例子中，M 是这两堆书的分割轴或者枢纽）。

还没开始排列就先分块解决问题的好处是，每本书只要找到它对应的那一堆就够了，你不需要考虑具体应该把它放在哪个位置上。 然后你可以进一步把这两堆再分成四堆：A 到 G，H 到 M，N 到 T，以及 U 到 Z。 如果再继续不停分块下去，最终会得到很多只有一本书的书堆（基础状态，不需要再为一本书如何排序而思考）, 并且如果你在分堆的时候注意摆放位置的话，这些一本书的书堆的次序，就是它们应该在书架上摆放的次序。 这个重复分堆的过程，就是快排序的工作原理。

在我们实现快排序代码的过程中，每次调用 quicksort() 都会将一堆元素（一个数组）传入进行排序。 同时会传入 left 和 right 两个参数来明确所分成的两堆的索引范围， 类似于二分搜索 binarySearch() 的 left、right 参数。 算法会选出一个中轴值，来与数组中的其他值进行比较，如果数组中的某个值比中轴值小，就会被放在左边一堆，比中轴值大，就会被放在右边一堆。这就是分块的步骤。 下一步，快排序函数 quicksort() 会被递归式的调用，来对这两个更小的数组进行再分割，直到分出了零个元素的数组。 这个列表就会随着递归调用，变得越来越有序，直到它彻底变为我们所需要的有序列表。

汉诺塔 Towers of Hanoi

汉诺塔是一个关于叠圆盘成塔的谜题。 这个谜题起始状态是，最大的圆盘在底部，并且越往上叠圆盘的尺寸越小。 圆盘的中心有小孔，因此可以被套在杆子上叠放在一起。 （译注：起始状态杆子共三根，并且像前面说的一样，圆盘按大小顺序套在同一根杆子上） 而谜题的目的，是将所有圆盘从一根杆子上移动到另一根杆子上，移动顺序遵循如下三条规则：

  1. 玩家一次只能挪动一个圆盘。
  2. 玩家所移动的圆盘只能是某根杆子最上面的一个。
  3. 玩家不能将大盘子叠在小盘子上面。
